# Module setup page — public IP/CIDR, defaults, consumer jumps

## Summary

Added FormSetup admin page for `websitepartials`. REST IP filtering stays on Dolibarr’s global `API_RESTRICT_ON_IP` (exact IPs). Public path gets module-owned `WEBSITEPARTIALS_PUBLIC_ALLOWED_IPS` with CIDR support. Also: default website ref, Cache-Control, and a 0..n consumer URL list for convenience jump links only.

## Changes

- `admin/setup.php` + `config_page_url` on descriptor; default consts on enable
- Lib: `websitepartials_ip_allowed` / CIDR helper + getters; `websitepartialsAdminPrepareHead`
- Langs, module README, ROADMAP P4 note, ChangeLog

## Decisions

- Do not duplicate REST IP allowlisting in the module
- Consumer URLs are not CORS/security controls
- Public IP helper ready for P1 `public/partial.php` to call

## Follow-ups

- P1 public router must call `websitepartials_ip_allowed(getUserRemoteIP())` and apply `websitepartials_cache_control()`
