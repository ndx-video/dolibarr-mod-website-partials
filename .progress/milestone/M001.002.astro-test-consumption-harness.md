# P1 — Astro test-consumption harness + island demo

## Summary

Added `dev/test-consumption/`: an Astro 5 (Node SSR) app that comprehensively exercises REST CRUD and public islands, plus a mini P3-shaped island rendering page.

## Changes

- Scaffold: `package.json`, `@astrojs/node`, env (`PARTIALS_BASE_URL`, `DOLAPIKEY`, website refs)
- Clients + ordered suite (`suite.ts`) covering status/auth, website/page CRUD, filters, slug rules, public json/html/404/400/no-PHP-exec
- UI: hub, harness (run/seed), demo (`set:html` body + fallback)
- API routes: `POST /api/run-suite`, `POST /api/seed`
- README pointer from repo README local-dev section

## Decisions

- Default `PARTIALS_BASE_URL` to `http://partials.gandalf.lan` so Node SSR avoids Caddy `tls internal` trust issues
- Destructive CRUD targets dedicated `WEBSITE_REF=demo-partials`; demo page uses `PUBLIC_WEBSITE_REF`
- Keep DOLAPIKEY server-side only
