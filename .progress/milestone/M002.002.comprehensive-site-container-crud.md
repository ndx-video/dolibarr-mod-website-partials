# P2 — Comprehensive site/container REST + module permissions

## Summary

Expanded `websitepartials` into a full DOLAPIKEY control plane for Website **sites** and all container `type_container` values, with module-owned nested permission toggles and DELETE. Stopped authorizing API calls via core `website` rights.

## Changes

- Declared 27 rights on `modWebsitePartials`: `website` + 8 container types × `read`/`write`/`delete`
- API: `GET/POST/PUT/DELETE websites`, type-gated container list/get/create/update/delete
- Lib loads `files.lib.php`, `website.lib.php`, `website2.lib.php` so core `Website::create` / `WebsitePage::delete` do not fatal
- Updated ROADMAP / READMEs / ChangeLog / AGENTS
- Smoke on `partials.gandalf.lan`: site CRUD, page CRUD, blogpost write → 403 without grant

## Decisions

- Keep URL segment `pages` for containers (compat); document as type-gated containers
- API auth uses only `websitepartials.*`; Website UI still uses core `website` rights
- Publish/unpublish remains `PUT` with `status`
