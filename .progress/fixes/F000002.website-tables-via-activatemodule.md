# Fix — Website tables missing after SQL-only enable

## Summary

`/admin/website.php` failed with `DB_ERROR_NOSUCHTABLE` / `relation "llx_website" does not exist` because `dev/scripts/up.sh` had flipped `MAIN_MODULE_WEBSITE=1` in `llx_const` without calling Dolibarr `activateModule()`, so `modWebsite::init()` never ran `_load_tables()`.

## Changes

- Deactivated/reactivated `modWebsite` via `activateModule` on the running stack — created `llx_website`, `llx_website_page`, `llx_website_extrafields`
- Added `dev/scripts/activate-module.php` helper
- Updated `dev/scripts/up.sh` to enable API, Website, and websitepartials through `activateModule` (and set `MAIN_FEATURES_LEVEL=2`)

## Decisions

- Const-only module enable is insufficient for modules that ship SQL; always use `activateModule` in the module-dev stack bootstrap

## Follow-ups

- None for this bug; P1 can create `main-website` / `welcome` against the new tables
